# FINAL STRICT AUDIT & REPAIR - MT5 BOT vs bobot2.py

Tugas:
Lakukan audit ulang secara ketat terhadap bot MT5 ini, membandingkan setiap detail dengan bobot2.py asli, lalu perbaiki hingga:
1. GUI identik 100% secara tampilan dan fungsi.
2. Semua fitur pre-start (user menentukan TP, SL, lot, strategi awal, news filter, dll) tersedia.
3. Semua sistem backend terhubung sempurna dengan GUI dan MT5.
4. Tidak ada error, lag, atau fungsi yang hilang.

## LANGKAH AUDIT

### 1. AUDIT GUI SECARA VISUAL & FUNGSIONAL
- Bandingkan layout, ukuran jendela, warna, font, icon, jarak antar elemen.
- Pastikan semua tab, panel, dan tombol ada sesuai bobot2.py.
- Pastikan setiap tombol memicu fungsi backend yang benar.
- Pastikan tabel posisi, panel statistik, kalkulator TP/SL, dan log viewer bekerja real-time.
- Lakukan koreksi langsung bila ada perbedaan.

### 2. TAMBAHKAN FITUR PRE-START SETTINGS
- Sebelum bot dijalankan, user harus bisa:
  - Menentukan TP (pips, harga, %, mata uang)
  - Menentukan SL (pips, harga, %, mata uang)
  - Memilih strategi awal
  - Mengatur lot size atau auto-lot
  - Mengaktifkan/menonaktifkan news filter
- Pastikan pengaturan ini langsung tersimpan dan digunakan strategi ketika bot mulai.

### 3. AUDIT KONEKSI & INTEGRASI
- Pastikan setiap aksi GUI → Backend → MT5 berjalan tanpa delay.
- Pastikan status koneksi MT5 selalu update di GUI.
- Pastikan semua pengaturan user di GUI diterapkan ke backend sebelum order pertama.

### 4. PERBAIKAN SISTEM DAN PENINGKATAN
- Zero error target (LSP, null reference, runtime error).
- Tambahkan validasi input user (contoh: TP/SL tidak boleh 0 atau negatif).
- Optimalkan performa GUI (minim CPU & RAM usage).
- Pastikan auto-reconnect MT5 berfungsi.
- Tambahkan logging ketika user mengubah setting pre-start.

### 5. PENGUJIAN FINAL
- Uji semua strategi setelah start.
- Uji pengaturan pre-start bekerja di semua strategi.
- Uji emergency stop langsung dari GUI.
- Uji news filter berfungsi.
- Uji export log & CSV.
